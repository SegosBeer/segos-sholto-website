Schritt für Schritt: Ein Bild ersetzen
1
Bild vorbereiten
Bild auf dem Computer in JPG oder PNG speichern. Empfehlung: auf squoosh.app komprimieren, sodass es max. 500 KB groß ist. Dateinamen ohne Leerzeichen und Umlaute wählen (z.B. brauerei-hero.jpg statt Brauerei Foto final.jpg).
2
Bild in den richtigen Ordner kopieren
Das Bild in den passenden Unterordner unter assets/images/ kopieren. Zum Beispiel ein Taproom-Bild nach assets/images/taproom/hero.jpg. Die LIES_MICH.md-Datei im Ordner zeigt welche Dateinamen erwartet werden.
3
HTML-Datei öffnen
Die zugehörige HTML-Datei in VS Code öffnen (z.B. pages/taproom.html). Mit der Tastenkombination Ctrl+F (Windows) oder Cmd+F (Mac) nach dem alten Bild-Link suchen.
4
Alten Bildlink ersetzen
Den alten langen Google-Link https://lh3.googleusercontent.com/... durch den neuen lokalen Pfad ersetzen. Zum Beispiel: ../assets/images/taproom/hero.jpg (für Seiten im pages/ Ordner) oder ./assets/images/home/hero.jpg (für die Startseite index.html).
5
Speichern & Testen
Datei speichern (Ctrl+S / Cmd+S). HTML-Datei im Browser öffnen (Doppelklick auf die Datei) und prüfen ob das Bild korrekt angezeigt wird.
Pfad-Schema: So wird ein Bildpfad geschrieben
Datei Pfad zum Bild Erklärung
index.html ./assets/images/home/hero.jpg Startet mit ./ (aktueller Ordner)
pages/taproom.html ../assets/images/taproom/hero.jpg Startet mit ../ (ein Ordner zurück)
pages/philosophie.html ../assets/images/philosophie/brauerei-1.jpg Startet mit ../ (ein Ordner zurück)
Achtung Groß-/Kleinschreibung beachten! Hero.jpg ist NICHT dasselbe wie hero.jpg. Am sichersten: Dateinamen immer komplett kleinschreiben.
Empfohlene Bildformate & Qualität
JPG — Für Fotos

Ideal für alle Fotos (Brauerei, Taproom, Atmosphäre). Qualität 80–85% reicht für Websites vollkommen aus. Max. 600 KB pro Bild.

HintergrundbilderAtmosphärePortraits

PNG — Für Produktbilder

Nur für Produktbilder (Dosen, Flaschen) die einen transparenten Hintergrund brauchen. PNG-Dateien sind größer, also sparsam einsetzen.

Kapitel Drei
Texte bearbeiten
— Grundregeln
Wie Text in HTML aussieht

HTML-Dateien haben Text der von spitzen Klammern umgeben ist. Du musst NUR den Text zwischen den Tags ändern, NICHT die Tags selbst.

SO SIEHT ES AUS: <h1 class="font-headline...">The Science of Soul.</h1> SO ÄNDERST DU DEN TEXT: <h1 class="font-headline...">Dein neuer Titel hier.</h1> NUR DEN INHALT ÄNDERN — NICHT DIE TAGS (<h1> und </h1>) ANFASSEN!
Die wichtigsten HTML-Elemente erklärt
HTML-Tag Bedeutung Beispiel
<h1> Haupt-Überschrift (groß, fett) "The Science of Soul."
<h2> Abschnitts-Überschrift "Current Batch"
<h3> Unter-Überschrift Produktname
<p> Fließtext / Beschreibung Längerer Beschreibungstext
<span> Kleiner Label-Text "Established 2024"
<a href="..."> Link (z.B. zum Shop) Shop-URL im href-Attribut
Suchen & Ersetzen — Der schnellste Weg
1
Datei öffnen, Suche öffnen
In VS Code die HTML-Datei öffnen. Dann Ctrl+H (Windows) oder Cmd+Option+F (Mac) drücken. Damit öffnet sich "Suchen & Ersetzen".
2
Alten Text eingeben
Im oberen Feld den Text eingeben den du ersetzen willst. Z.B. The Science of Soul.
3
Neuen Text eingeben & ersetzen
Im unteren Feld den neuen Text eingeben. Dann "Ersetzen" oder "Alle ersetzen" klicken. Speichern nicht vergessen!
Niemals anfassen Alles was mit class="...", href="...", src="...", onclick="..." oder id="..." anfängt — das sind Styling- und Funktions-Anweisungen. Nur den sichtbaren Text dazwischen ändern.

Neues Produkt
hinzufügen
So fügst du ein neues Bier zur Sortimentsseite hinzu
1
Produktbild vorbereiten
Produktfoto als PNG (transparenter Hintergrund) erstellen. Empfohlen: 400 × 600 Pixel. Datei benennen z.B. neues-bier-dose.png und in assets/images/segos-sortiment/ (oder sholto-sortiment) kopieren.
2
Sortimentsseite öffnen
pages/segos-sortiment.html in VS Code öffnen. Eine bestehende Produktkarte komplett kopieren (den gesamten Block von <div class="...card..."> bis zum schließenden </div>).
3
Kopie einfügen
Die kopierte Produktkarte direkt nach der letzten Karte einfügen. Danach die Texte (Name, Sorte, ABV, Beschreibung) und den Bildpfad (src="...") mit den neuen Daten befüllen.
4
Speichern & testen
Datei speichern, im Browser öffnen und prüfen ob die neue Karte korrekt angezeigt wird.
Tipp: Produktkarten-Vorlage Eine vollständige Produktkarte sieht in etwa so aus. Alles in GROSSBUCHSTABEN sind die Stellen die du ersetzen musst:
<div class="bg-surface-container p-5 md:p-8 flex flex-col justify-between group"> <div> <h3 class="font-headline font-black text-2xl uppercase">PRODUKTNAME</h3> <p class="font-body text-primary/60 mt-2 uppercase text-[10px]">SORTE | ABV%</p> </div> <img class="w-full h-48 object-contain my-8" src="../assets/images/segos-sortiment/DATEINAME.png"/> <button class="w-full bg-primary text-white py-4 font-label font-bold uppercase text-xs">In den Warenkorb</button> </div>

Workflow: Änderungen online stellen
Wie die Website gehostet wird Die Website liegt auf GitHub (Versionskontrolle) und wird von dort aus automatisch auf den Webserver (z.B. Netlify, Vercel oder GitHub Pages) veröffentlicht. Jede Änderung die du auf GitHub hochlädst, erscheint automatisch auf segos.beer.
1
Änderungen lokal testen
Bevor du etwas online stellst: HTML-Datei im Browser öffnen (Doppelklick) und alles prüfen. Bilder sichtbar? Texte korrekt? Links funktionieren?
2
GitHub Desktop öffnen
GitHub Desktop starten (kostenlos unter desktop.github.com). Das Repository "segos-sholto-website" auswählen. Alle geänderten Dateien werden links angezeigt.
3
Commit erstellen
Unten links einen kurzen Beschreibungstext eingeben, z.B. "Taproom Öffnungszeiten aktualisiert" oder "Neues Produktbild IPA". Dann "Commit to main" klicken.
4
Push to origin
Oben rechts "Push origin" klicken. Die Änderungen werden zu GitHub hochgeladen. Innerhalb von 1–2 Minuten ist die Website aktualisiert.
5
Live-Website prüfen
segos.beer im Browser öffnen (Ctrl+Shift+R für Hard-Reload) und prüfen ob die Änderungen sichtbar sind.
Häufige Probleme & Lösungen
Problem Mögliche Ursache Lösung
Bild wird nicht angezeigt Falscher Dateipfad oder falscher Dateiname Pfad und Dateinamen genau prüfen (Groß-/Kleinschreibung!)
Seite sieht kaputt aus Versehentlich ein HTML-Tag gelöscht Änderung rückgängig machen (Ctrl+Z), nochmal versuchen
Änderung nicht sichtbar Browser zeigt gecachte Version Hard-Reload: Ctrl+Shift+R (Win) / Cmd+Shift+R (Mac)
Website lädt sehr langsam Bilder zu groß (über 1 MB) Bilder auf squoosh.app komprimieren (Ziel: unter 400 KB)
Age-Gate erscheint immer wieder Browser-Cookies/Storage gelöscht Normal — nach Bestätigung bleibt es für 7 Tage gespeichert
Nützliche Tools (alle kostenlos)
squoosh.app

Bilder komprimieren — einfach Bild hochladen, Qualität einstellen, herunterladen. Spart 70–80% Dateigröße ohne sichtbaren Qualitätsverlust.

VS Code

Kostenloser Code-Editor von Microsoft. Perfekt für HTML-Bearbeitung. Zeigt Syntaxfehler sofort an.

GitHub Desktop

Einfaches Programm zum Hochladen von Änderungen auf GitHub — ohne Kommandozeile.

remove.bg

Hintergrund von Produktfotos automatisch entfernen (für PNG-Produktbilder mit transparentem Hintergrund). Kostenlose Version für einzelne Bilder.
